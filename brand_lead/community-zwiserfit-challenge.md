## 挑战赛定位

**核心理念：** "不写白皮书，写可运行的代码。不做PPT，做可复现的解决方案。"

ZWISERFIT Challenge 不是黑客松——它是**分布式协作调试**。每一期聚焦一个真实的技术难题，任何人可以在自己的环境里复现、解决、提交。最优方案直接合入主仓库。

**区别于传统黑客松：**
- ❌ 不做48小时熬夜冲刺
- ❌ 不做PPT路演
- ✅ 异步参与，2周周期，全球时区友好
- ✅ 只评价可运行的代码，不评价演示文稿
- ✅ 获奖方案直接进入生产环境

## Challenge #1: "第一公里" — ZWF-20 数据探针（5月11日-25日）

**主题：** 用最少的代码，从ZWF-20人脸识别终端获取一条打卡数据

**背景：** 我们有一台ZWF-20人脸机，会员可以在上面刷脸开门。但服务器收不到任何数据。我们不知道它的通信协议。我们需要一个"探针"。

**挑战任务：**
1. 研究ZWF-20设备的通信协议（RS-485？HTTP？私有协议？）
2. 写一段不超过200行的Python脚本
3. 从设备获取任意一条数据（用户ID、打卡时间、事件类型）
4. 输出为标准JSON格式

**无需硬件参与方式：**
- 如果无法接触ZWF-20设备，可以用模拟器/模拟数据
- 协议分析文档同样计入评判

**评分标准：**
| 维度 | 权重 | 说明 |
|------|:---:|------|
| 可运行性 | 40% | 代码能跑，产出真实JSON |
| 文档质量 | 30% | 协议分析/复现步骤清晰 |
| 代码优雅 | 20% | 简洁、有注释、有错误处理 |
| 创新性 | 10% | 是否有意外的好思路 |

**奖励：**
- 🥇 **金牌探针手：** 方案合入主仓库 + README致谢 + ZWISERFIT Contributor T恤（待实物化）
- 🥈 **银牌探针手：** GitHub Sponsor 致谢 + 社区公告表彰
- 🥉 **铜牌探针手：** Issue 内公开致谢
- 所有有效提交者：列入 CONTRIBUTORS.md

**参与方式：**
1. Fork `ZWISERFIT/zwiserfit-ai-store-manager`
2. 在 `contrib/challenge-001/` 目录下提交你的方案
3. 包含：代码文件 + README.md（解释你的方案）
4. 开 PR，标题以 `[CHALLENGE-001]` 开头
5. 截止时间：2026-05-25 23:59 UTC+8

## Challenge #2: "反脑补" — AI数据源预检系统（5月25日-6月8日，预告）

**主题：** 设计一个AI Agent在生成报告前必须通过的"数据源预检"

**背景：** ZWISERFIT有5个AI Agent，每个有明确的能力边界。但目前没有程序化机制阻止Agent报告它无权访问的数据。

**挑战任务：**
1. 将 `Agent数字孪生档案.md` 转换为机器可读的 JSON schema
2. 实现一个 Python 预检函数：输入 agent_id + data_source → 返回 ACTIVE/INACTIVE/TEST-ONLY
3. 提供集成示例：至少1个Agent演示预检拒绝流程

**更多细节：** 见 `HALLU-001` Issue
