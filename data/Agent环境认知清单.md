## 四、任务预检强制流程

执行任何工具生成任务前，必须执行以下步骤：

```
0. 调取个人数字孪生档案 → data/digital-twins/{你}-dt.md
   - 检查：物理环境限制、工具集、可用API、模型参数
   - 确认：当前能力边界是否覆盖任务需求

1. 读本文件 → 确认所需工具是否在可用清单中
2. 如果工具不在清单中：
   a. 检查是否有替代方案
   b. 优先选择纯文本输出（Markdown）
   c. 优先选择成熟API（非本地生成）
3. 如果必须安装新工具：
   a. 先上报 Shuyu/Tristan 审批
   b. 安装后更新本清单 + 更新个人数字孪生档案
4. 设置探索预算（如果是探索型任务）：
   - 默认预算：¥0.50 / 10分钟 / ≤3次尝试
   - 详见《军团执行章程》§2
5. 禁止行为：
   ❌ 暴力穷举尝试不可用方案
   ❌ 假设GPU可用或尝试CUDA方案
   ❌ 使用 canvas 等已验证不稳定的工具进行图像生成
   ❌ 连续3次失败后不汇报、继续尝试
   ❌ 跳过预检直接执行
```

### 4.1 预检三问（快速自查）

```
Q1: 我清楚当前环境的物理限制吗？
    → 查阅数字孪生档案 §1 + Agent环境认知清单

Q2: 我有完成此任务的工具集吗？
    → 查阅数字孪生档案 §2 + TOOLS.md

Q3: 我的模型参数适合此任务吗？
    → 查阅数字孪生档案 §4 + 故障档案 §2
```

## 五、失败上报标准

任务失败后，必须在 **3 分钟内** 向上级提交包含以下内容的失败报告。

### 5.1 探索预算耗尽上报

在遵循《军团执行章程》第二章「探索成本红线」的前提下：
- 同一任务连续失败 **3 次** → **必须立即停止并上报**
- 探索预算耗尽（默认为¥0.50/10分钟/3次尝试）→ **暂停→上报→等待决策**
- 严禁在预算耗尽后继续穷举尝试

### 5.2 标准失败报告模板

```
[失败上报]
- 任务：<原始任务描述>
- 尝试方案：<逐一列出，标注每步结果>
- 失败根因：<技术原因>
- 环境约束：<是否有不可逾越的硬件/软件限制>
- 建议下一步：<替代方案或请求上级决策>
```

**禁止：** 含糊其辞（"不太行"、"好像有问题"）、沉默、跳过上报

## 六、交付标准（Shuyu式严谨交付）

参考《军团执行章程》第四章，所有交付物必须包含：

```markdown
【设计依据】
- 方案选择：<为什么选方案A而非B>
- 理由：<引用环境清单/故障档案/过往案例>
- 参考来源：<文件名+章节号>

【可优化空间】
- 已知限制：<当前交付物的局限>
- 改进方向：<如果资源升级/模型升级可达到的效果>
- 请求反馈：<具体的确认问题>
```
