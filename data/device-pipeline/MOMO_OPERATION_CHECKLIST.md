# 第1步：在门禁设备上录入会员人脸

## 1.1 准备工作

- [ ] 确认设备已通电、屏幕亮起（显示时间/摄像头画面）
- [ ] 确认设备已联网（屏幕右上角通常有网络图标）
- [ ] 准备好要录入的会员：**姓名 + 会员编号**（例如 MEMBER_001）

## 1.2 进入管理员菜单

| 操作 | 说明 |
|------|------|
| ① 在主屏幕 **长按** 屏幕中央 3 秒 | 弹出登录界面 |
| ② 输入管理员密码 | 默认密码：**`888888`**（若不对，试 `123456` 或 `000000`） |
| ③ 点击 **「确定」** | 进入主菜单 |

> 💡 如果设备没有弹出密码框，而是直接进入菜单，说明管理员密码已被清除，可以直接操作。

## 1.3 录入人脸（核心操作）

进入主菜单后，按以下顺序点击：

```
主菜单 → [用户管理] → [新增用户] → [输入用户ID]
```

### 用户ID输入规则

**重要：** userId 不能重复。请按以下规则输入：

| 会员编号 | 请输入设备上的 「用户ID」 |
|----------|--------------------------|
| MEMBER_001 | `1` |
| MEMBER_002 | `2` |
| MEMBER_003 | `3` |
| ... | 以此类推 |
| MEMBER_050 | `50` |

> 如果某个编号已被占用，就往后顺延（比如 MEMBER_001 用 `1`，MEMBER_002 用 `2`……按顺序分配）。

### 录入人脸步骤

```
① 输入用户ID（数字）       →   点击「确定」
② 输入用户姓名（中文）     →   点击「确定」（可选，跳过也行）
③ 选择 [人脸登记]          →   面对摄像头
④ 听到「滴」声或看到绿色框 →   登记成功 ✅
⑤ 点击 [保存]              →   完成
```

### 注意事项

- ⚠️ **一次只录一个人**，录完确认再录下一个
- 📸 站在设备正前方，光线充足，不要戴帽子/口罩
- 🆔 **用户ID 记下来！** 下一步要用的

# 第2步：获取设备端 userId

录入完成后，需要查出刚才录入的 userId 数字是多少。

## 方法A：在设备屏幕上查看（推荐）

```
主菜单 → [用户管理] → [用户列表] → 找到刚才录入的人名
```

屏幕上会显示该用户的详细信息，其中 **用户ID（或 UserID）** 就是你要的数字。

## 方法B：通过设备功能键查看

部分型号支持在主界面直接操作：

```
主屏幕 → 点击右上角 [菜单] 图标 → [用户管理] → [用户列表]
```

## 方法C：通过服务器远程查看

如果设备已联网且我们后台在运行，访问以下链接查看注册人员列表：

```
http://localhost:5000/api/v1/users?device_id=ZWF-20-001
```

> 这需要我在后台操作，你告诉我设备名称和录入的姓名即可。

## 拿到 userId 后要干什么？

**写下来，贴在看板上——** 格式如下：

| 门店 | 会员编号 | 设备端 userId | 录入日期 |
|------|---------|-------------|---------|
| XX店 | MEMBER_001 | `1` | 2026-04-30 |
| XX店 | MEMBER_002 | `2` | 2026-04-30 |

# 第3步：将 userId 回填到映射表

这一步需要把设备端 userId（你刚才记下的数字）填写到 `member-device-mapping.json` 文件中，让系统知道「设备上的用户1 = 会员MEMBER_001」。

## 方法A：通过管理员界面回填（推荐，不需技术）

把以下信息发给我（Tristan）：

```
认领请求：
- 设备端 userId：______（你记下的数字）
- 会员编号：MEMBER_XXX（例如 MEMBER_001）
- 门店：XX店
- 录入时间：2026-04-30
```

我在后台执行以下 API 将其写入系统：

```bash
curl -X POST http://localhost:5000/api/v1/mapping/register \
  -H "Content-Type: application/json" \
  -d '{"device_user_id": "1", "member_id": "MEMBER_001", "updated_by": "店长姓名"}'
```

之后该会员的打卡记录会自动匹配并生成分析数据。

## 方法B：直接编辑 JSON 文件（紧急情况下）

如果联系不上我，可以请 Ethan 或其他技术同事帮忙：

1. 用 **记事本**（Windows）或 **文本编辑**（Mac）打开文件：

   ```
   /home/agentuser/shared/member-device-mapping.json
   ```

2. 找到对应的会员，例如 MEMBER_001：

   ```json
   "MEMBER_001": {
       "name": "胡**",
       "phone": "136****0313",
       "gender": "男",
       "device_id": null,     ← 把 null 改成数字
       ...
   }
   ```

3. 将 `"device_id": null` 改成 `"device_id": "1"`（用你记下的 userId）：

   ```json
   "MEMBER_001": {
       "name": "胡**",
       "device_id": "1",      ← 已修改
       ...
   }
   ```

4. 保存文件，然后执行映射重载：

   ```bash
   curl -X POST http://localhost:5000/api/v1/mapping/reload
   ```

5. 验证是否生效：

   ```
   返回: {"total_members": 118, "bound_members": 1}
   ```

② 拿 userId 回填映射表（方法 A 或 B）
③ 让这 3 位会员在门禁前刷脸开门
④ 门禁验证通过 ✅ → 后台自动记录打卡数据
⑤ 我验证标准化 JSON 是否生成

## 常见问题

| 问题 | 解决方法 |
|------|---------|
| 密码不对 | 试 `888888` / `123456` / `000000` / `admin` / `1234`。都不对就联系设备供应商重置 |
| 屏幕不灵敏 | 用指尖按压，不要用指甲 |
| 提示「已存在用户」 | 说明这个 userId 已被占用，换一个没用过的数字（如 `100` 往后） |
| 人脸登记失败 | 检查光线是否充足，面部是否正对摄像头，距离是否 30-50cm |
| 设备死机 | 拔电源等 10 秒再插上重启 |
